# NewsControl

보도기술팀 업무 관리 시스템. Express 서버(`server/`)와 Vite + React 클라이언트(`client/`)로 구성된 모노레포입니다.

## 문서 모음

```bash
npm run doc
```

후 `http://localhost:4400/doc/index.html` 접속 — README·PRD·[개발 기록(DEVLOG)](doc/DEVLOG.md)을 검색 가능한 화면으로 볼 수 있습니다. 개발 흐름·문제·해결 방법은 기능이 바뀔 때마다 `doc/DEVLOG.md`에 이어서 기록합니다.

## 실행방법

### 1. 의존성 설치 (최초 1회)

```bash
npm run install-all
```

루트, `server/`, `client/`의 의존성을 한 번에 설치합니다.

### 2. 개발 서버 실행

```bash
npm run dev
```

`server`(Express, nodemon)와 `client`(Vite)를 동시에 실행합니다.

- 서버: http://localhost:5001
- 클라이언트: http://localhost:5173

브라우저에서 http://localhost:5173 접속하면 됩니다. 로컬 개발 모드에서는 `data/` 폴더의 JSON 파일을 직접 읽고 쓰며, Vercel Blob 연동은 배포 환경(`VERCEL` 환경변수가 설정된 경우)에서만 사용됩니다.

### 3. 프로덕션 빌드 (선택)

```bash
npm run build --prefix client
```

빌드 결과물(`client/dist`)은 로컬 모드에서 서버가 정적 파일로 함께 서빙합니다.

## 전광판(`?display`) 표시 기준

`http://localhost:5173/?display` (또는 대시보드의 "전광판" 버튼)로 여는 전광판 화면은 두 영역으로 구성됩니다. 관련 코드: `client/src/App.tsx`의 `isDisplayMode` 블록.

### 1. 상단 한줄공지 바 (1분마다 다음 줄로 스크롤)
- 조건: `announcement.expiresAt > 현재시각` (아직 만료 안 됨)
- 활성 한줄공지가 여러 개면 한 번에 한 줄만 표시하고, 1분마다 다음 공지로 자동 전환됨(마지막이면 처음으로 순환) — `Math.floor(현재시각 / 60000) % 활성개수`로 현재 몇 분인지에 따라 표시할 공지를 결정
- 전환 시 아래에서 위로 슬라이드-인 되는 스크롤 애니메이션 적용
- `expiresAt`은 등록 시 고른 기간(3일 / 1주)만큼 현재시각에 더해서 결정됨

### 2. 메인 카드 그리드 (최대 4칸)
후보 조건:
1. `showFrom`(예약 공개 시각)이 설정돼 있고 아직 도래하지 않았으면 제외
2. **공지일(표시 기간)** 안에 오늘이 포함돼야 함 — 예정일(`date`)부터 `duration`(1일/3일/7일, 기본 1일)만큼의 날짜 범위. 현재 시각이 오후 8시 이후면 내일 날짜가 그 범위에 포함되는 공지도 후보에 포함
3. `duration`이 기본값(1일)인 공지는 예정 시각이 지나면 후보에서 즉시 제외됨(기존 동작) — 3일/7일처럼 기간을 지정한 공지는 예정 시각이 지나도(완료 후에도) 남은 기간 동안 계속 후보로 유지됨
4. `isPinned`(중요 표시) 여부는 후보 선정에 영향 없음 — 빨간 강조 스타일만 결정

**슬롯 채우기 규칙**
- 전광판이 비어 있으면(슬롯 0개): 후보 중 표시 순서가 빠른 것부터 최대 4개를 바로 채움 (아직 예정 시각이 안 지난 공지는 가까운 순, 기간만으로 남아있는 지난 공지는 최근에 지난 것부터 그 뒤에)
- 슬롯이 이미 차 있으면: 이미 표시 중인 공지는 더 이상 후보가 아니게 될 때까지(1일짜리는 예정 시각이 지날 때, 기간 지정 공지는 표시 기간이 끝날 때) 슬롯을 유지(sticky)하며, 빈 자리는 위와 같은 순서로 채움
- 이미지가 첨부된 공지는 텍스트 대신 이미지만 크게 표시
- 관련 코드: `App.tsx`의 `isNoticeVisibleOnDay()`, `compareNoticeDisplayOrder()`, `NOTICE_DURATION_DAYS`

**카드 강조 스타일 우선순위**: 깜빡임(빨강, 임박 1회성) > 중요/pinned(빨강) > 임박 알람(노랑) > 지난 공지(연노랑, "완료" 배지) > 기본

### 3. 임박 알람 (진한 노란색)
설정(`설정 > 공지 알림 타이밍`)의 **"노란색 강조 (분 전)"** 값(기본 30분)을 기준으로, 공지 예정 시각까지 남은 시간이 그 값 이하로 들어오면 알람이 시작됩니다.

- 10분 간격으로 1분간만 진한 노란색 배경으로 깜빡임 (예: 기본값 30분 기준 → 예정 시각 30분 전, 20분 전, 10분 전에 각 1분씩 알람)
- 예정 시각이 지나면(0분 이하) 알람은 멈추고 "지난 공지" 스타일로 전환됨
- 관련 코드: `App.tsx`의 `isInAlarmWindow()`, `ALARM_INTERVAL_MINS`(10), `ALARM_DURATION_MINS`(1), CSS는 `App.css`의 `.dm-card.dm-alarm`
