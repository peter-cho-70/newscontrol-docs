# DEVLOG — NewsControl 개발 기록

> 보도기술팀 업무 관리 시스템의 개발 흐름, 마주친 문제와 해결 방법을 시간순으로 기록한다.
> 새 기능을 추가하거나 문제를 해결할 때마다 이 문서 최상단(타임라인 맨 앞)에 새 항목을 이어서 기록한다.
> 보기 좋은 화면으로 읽으려면 `npm run doc` 실행 후 `http://localhost:4400/index.html` → 개발 기록.

## 프로젝트 개요

- **목적**: 보도기술팀(TD/VD/AD/LD/SD)의 업무 노하우를 데이터베이스화하고, 전광판 공지 시스템으로 스튜디오 현장에 실시간 안내를 제공 (`PRD.md` 참고)
- **구성**: 모노레포 — `client/`(Vite + React + TypeScript), `server/`(Express), `data/`(JSON 시드 데이터)
- **배포**: Vercel (`vercel.json` — `client`는 `@vercel/static-build`, `server`는 `@vercel/node`)
- **데이터 저장**: 로컬 개발 시 `data/initial_work_guide.json` 파일을 직접 읽고 쓰며, 배포(서버리스) 환경에서는 Vercel Blob에 저장

## 타임라인

### 1. 초기 커밋 (2026-06-22, `46f310a`)
Vite+React 프런트엔드와 Express 백엔드로 기본 골격 구성. 역할별 업무 가이드, 프로그램별 주의사항 등 PRD의 Phase 1~2에 해당하는 정적 구조를 구현.

### 2. Vercel 배포 초기 트러블슈팅 (2026-06-22)
로컬에서는 문제없이 돌아가던 앱이 Vercel에 올리자 연쇄적으로 깨졌다. 세 번의 커밋에 걸쳐 원인을 하나씩 제거했다.

| 커밋 | 문제 | 원인 | 해결 |
|---|---|---|---|
| `b15b2f7` | 업로드 라우트 호출 시 모든 요청이 500 에러 | `require('sharp')`를 모듈 최상단에서 호출 → Vercel 서버리스 런타임의 네이티브 바이너리 로딩 문제로 **모듈 로드 자체가 크래시**, 그 라우트를 쓰지 않는 다른 요청까지 전부 죽임 | 업로드 핸들러 **내부에서 지연 로딩(lazy require)** 하고, 실패해도 썸네일 생성만 건너뛰도록 try/catch로 감쌈 |
| `aed0241` | SPA 루트 경로(`/`)가 404 | `vercel.json`의 라우팅이 filesystem-handle 방식이 아니어서 정적 라우트보다 우선순위가 꼬임 | `vercel.json` 라우팅 방식을 filesystem 우선 처리로 변경 |
| `4c48111` | 정적 자산(`/assets/*`, favicon, icons)이 404 | `@vercel/static-build`는 빌드 결과물을 `dist/` 밑이 아니라 **빌드 `src` 디렉터리 경로(`client/`) 그대로** 배포하는데, 번들된 HTML은 루트 상대 경로(`/assets/...`)를 참조 | `vercel.json`에 `/assets/(.*)`, `/favicon.svg`, `/icons.svg`를 `client/...`로 명시적으로 rewrite하는 라우트 추가 |

**교훈**: `@vercel/static-build` + 모노레포 조합에서는 "로컬 빌드 결과물 경로 = 배포 후 서빙 경로"라는 가정이 깨진다. 새로운 정적 자산 경로를 추가할 때마다 이 매핑을 다시 점검해야 한다.

### 3. UI 리디자인 (2026-06-22, `7871c89`)
어두운 남색 SaaS 톤 팔레트를 종이 질감의 따뜻한 색감 + 세이지그린 포인트로 교체. Pretendard 웹폰트 적용으로 한글 타이포그래피 개선. 이때까지는 순수 프런트엔드 스타일 작업이라 배포 이슈는 없었음.

### 4. 공지 데이터 영속성 문제 (2026-06-22~07-02, `769a75b`)
**문제**: 서버리스 환경에서 공지사항을 `/tmp`에 쓰고 있었는데, 서버리스 함수는 콜드스타트/재배포마다 인스턴스가 새로 뜨므로 `/tmp` 내용이 날아간다 → 방금 추가한 공지가 사라지는 현상 발생.

**해결**: 데이터를 **Vercel Blob**에 저장하도록 전환 (`server/index.js`의 `BLOB_KEY = 'newscontrol-guide.json'`). 추가로, blob 조회가 일시적으로 실패한 경우와 "blob이 아직 존재하지 않는 최초 상태"를 구분해서, **일시적 읽기 에러일 때는 시드 데이터로 덮어쓰지 않도록** 처리 — 안 그러면 네트워크 hiccup 한 번에 실제 데이터가 시드 데이터로 롤백되는 사고가 날 수 있었음.

같은 커밋에서 UX 개선도 다수 포함:
- 대시보드에 오늘의 한줄공지 + 전광판 실시간 미리보기 노출
- 전광판 슬롯을 "sticky"하게 유지 — 다가오는 공지가 시간 되기 전에 화면에서 밀려나지 않게
- 공지 카드 텍스트 오버플로우 수정, 높이 제한 + 내부 스크롤
- 리허설처럼 반복되는 일정을 위해 퀵등록에서 날짜/시간 슬롯 최대 3개(10분 단위) 지원

### 5. 위치 프리셋 · 이미지 첨부 · 날짜 버그 (2026-07-02, `a317c52`)
**문제**: 날짜 입력 기본값을 `toISOString()`으로 채웠는데, 이는 **UTC 기준 문자열**이라 한국 새벽 시간대(KST 자정~오전 9시 사이)에는 하루 전 날짜가 표시되는 버그가 있었음.

**해결**: 로컬 타임존 기준으로 날짜를 포맷하는 헬퍼로 모든 날짜 입력 초기화 지점을 교체.

그 외:
- 공지 추가/수정 폼에 위치 프리셋 칩(뉴스센터A/B, 스포츠, 사무실, 기타) — 자유 입력도 유지
- 공지에 이미지 첨부 지원, 전광판에서는 이미지가 있으면 텍스트 카드 대신 이미지만 표시
- 기존 공지 수정 시 날짜/시간도 변경 가능하도록 개선

### 6. 전광판 알람/스크롤/공지일 (2026-07-02, `0a5b835`)
전광판(`?display`) 표시 로직을 상세화:
- **임박 알람**: 설정한 "노란색 강조" 시간(기본 30분 전) 이내로 들어오면, 10분 간격으로 1분씩 진한 노란색으로 깜빡이는 알람 표시
- **한줄공지 스크롤**: 활성 한줄공지가 여러 개면 1분마다 자동으로 다음 줄로 슬라이드 전환
- **공지일(표시 기간)**: 일반 공지에도 1일/3일/7일 표시 기간을 지정할 수 있게 해서, 예정 시각이 지나도 지정 기간 동안은 계속 노출
- 이미지 붙여넣기(Ctrl+V)로도 첨부 가능, 텍스트 없이 이미지만으로도 등록 가능
- README에 전광판 표시 규칙을 상세히 문서화 (`README.md` 참고 — 슬롯 채우기 규칙, 카드 강조 우선순위 등)

### 7. 개발 문서 체계 정리 (2026-07-11)
루트에 있던 `DEVLOG.md`를 `doc/DEVLOG.md`로 옮기고, `autoplay` 프로젝트의 `doc/index.html` + `doc/viewer.html` 패턴을 그대로 가져와 문서 허브를 구성. 외부 라이브러리 없이 동작하는 자체 마크다운 렌더러(표/코드블록/체크박스/TOC 지원)와 라이트/다크 테마 토글을 그대로 재사용. `npm run doc`로 로컬 정적 서버를 띄워 `doc/index.html`에서 README·PRD·DEVLOG를 검색·열람할 수 있게 함.

## 지금까지의 핵심 교훈 정리

1. **Vercel + 모노레포 정적 빌드**: `@vercel/static-build`의 산출물 경로는 직관과 다르다. 새 정적 자산을 추가하면 `vercel.json` 라우트도 같이 점검할 것.
2. **네이티브 모듈은 지연 로딩**: 서버리스 런타임에서 `sharp` 같은 네이티브 의존성은 모듈 최상단에서 요구하면 전체 함수가 죽는다. 실제로 쓰는 시점에 로드하고 실패를 격리할 것.
3. **서버리스 = 상태 없음**: `/tmp` 등 로컬 파일시스템에 의존하는 쓰기는 콜드스타트마다 사라진다. 영속 데이터는 반드시 외부 스토리지(Blob 등)로.
4. **일시적 에러 ≠ 데이터 없음**: 외부 스토리지 읽기 실패 시 무조건 시드값으로 폴백하면 실데이터가 유실될 수 있다. "존재하지 않음"과 "읽기 실패"를 구분해서 처리.
5. **날짜는 항상 로컬 타임존 기준으로**: `toISOString()`은 UTC라서 한국 새벽 시간대에 날짜가 하루 밀리는 버그를 만든다.
